What Already Exists

1. Pixel Agents (VS Code Extension)

Source: Open-source GitHub project with 7.8k stars

What it does:

Turns Claude Code AI agents into pixel-art office characters

Each agent becomes an animated character with real-time activity reflection

Characters walk around, sit at desks, show speech bubbles when waiting

Built-in office layout editor with furniture, floors, walls

Tracks agent status by watching JSONL transcript files (purely observational)

Tech Stack:

Extension: TypeScript, VS Code Webview API, esbuild

Webview: React 19, Canvas 2D, Vite

Game loop with BFS pathfinding and state machine (idle → walk → type/read)

Limitations:

Currently VS Code only, tied to Claude Code terminals

Agent-terminal sync can desync with rapid open/close

Status detection is heuristic-based (idle timers, not perfect)

Key Insight: The architecture is being redesigned to be platform-agnostic and agent-agnostic through composable adapters.

2. OpenClaw Dashboard (Go-based)

Source: mudrii/openclaw-dashboard — 443 stars, production-ready

What it does:

Single-page command center for OpenClaw installations

12 dashboard panels: metrics, health, costs, cron jobs, sessions, token usage, sub-agents, charts, chat

Zero external dependencies (single Go binary + embedded HTML/CSS/JS)

Real-time monitoring with auto-refresh every 60 seconds

AI chat panel that queries your live data using OpenClaw gateway

Tech Stack:

Backend: Go (single binary)

Frontend: Pure HTML/CSS/JS/SVG — no React, no build step, ~6k lines

Runs as system service (launchd/systemd)

Features:

Top metrics bar with CPU/RAM/Swap/Disk color-coded by thresholds

Agent hierarchy tree showing parent → sub-agent relationships

Per-model cost breakdown with 7d/30d/all-time tabs

Gateway runtime probes (liveness, readiness, failing deps)

6 built-in themes (3 dark, 3 light)

Key Insight: This is the functional analytics layer — it gives you what's happening but not the visual character simulation.

3. Agent Office (VS Code Extension)

Source: YouTube demo, GitHub harishkotra/agent-office

What it does:

Takes Pixel Agents concept and adds autonomous behavior

Agents hire interns, assign tasks to each other, execute code

Fully local running on Ollama multi-agent office

Custom layout designer

Key Insight: This is the "self-growing teams" vision — agents hiring agents, but it's conceptual and less stable than Pixel Agents.
