# ReClaw 2.0 — Session Summary
### What We Fixed, Why It Matters, What's Unlocked

---

## The Problem We Started With

ReClaw 2.0 was fully built — agents, memory, vault, Clawsmith forge, dashboard — but **Ollama showed as "off" everywhere.** That one broken connection meant:

- Agents couldn't think (no LLM brain)
- Memory/vault couldn't embed (no semantic search)
- Tools couldn't generate responses
- The whole stack was structurally sound but functionally dead

The root cause: **inside a Docker container, `127.0.0.1` points back to the container itself — not your host machine.** Ollama runs on the host. So every call to `http://localhost:11434` from inside ReClaw looped back into an empty container and got nothing.

---

## What We Changed

### 1. `docker-compose.yml` — Ollama Bridge

Added `extra_hosts` to both services:

```yaml
extra_hosts:
  - "host.docker.internal:host-gateway"
```

This injects a `/etc/hosts` entry inside every container that maps `host.docker.internal` to the actual host machine's gateway IP — automatically, regardless of what that IP is.

Added environment variables so ReClaw always uses the right address:

```yaml
environment:
  - OLLAMA_BASE_URL=http://host.docker.internal:11434
  - OLLAMA_HOST=http://host.docker.internal:11434
```

**Result:** Every service in the stack — agents, tools, memory, API — now routes Ollama calls to the real host instead of looping back to itself.

### 2. `dashboard/index.html` — Live Ollama Status Bar

Added a status bar at the top of the Agent Floor dashboard that:

- Polls `http://host.docker.internal:11434/api/version` every 30 seconds
- Shows `● ON` in green with Ollama version number when reachable
- Lists all pulled models (`/api/tags`)
- Shows `✖ OFF` in red with exact fix instructions when unreachable
- Has a manual `↺ REFRESH` button

No more SSH + curl to check if Ollama is alive. Open the dashboard URL and you know instantly.

### 3. `fix-ollama.sh` — One-Shot Repair Script

A single script that:
1. Patches Ollama's systemd service to bind on `0.0.0.0:11434` instead of just `127.0.0.1`
2. Reloads and restarts Ollama
3. Tests the connection from the host gateway
4. Brings the Docker stack back up

For any future server rebuild or new machine — one `sudo bash fix-ollama.sh` and everything is wired.

### 4. Port Conflict Resolution

Discovered OpenClaw gateway already owns port `8000`. Moved ReClaw API to `8001` so both run simultaneously without conflict:

| Service | Port | Purpose |
|---|---|---|
| OpenClaw Gateway | `:8000` | Agent orchestration, skills, plugins |
| ReClaw API | `:8001` | ReClaw 2.0 custom agent API |
| ReClaw Dashboard | `:8080` | Visual Agent Floor + Ollama status |
| Ollama | `:11434` | Local LLM inference (host) |

## What This Unlocks

Every integration that was wired but broken now works:

| What | How it uses Ollama | What you gain |
|---|---|---|
| **ReClaw Agents** | `OLLAMA_BASE_URL` env var already in code | Agents actually execute LLM calls — Researcher, Analyst, Grant Watcher all have a brain now |
| **Clawsmith Forge** | `clawsmith.py` calls Ollama for blueprint generation | Blueprint → human gate → forge flow works end to end |
| **Memory + Vault** | `/api/embeddings` via `nomic-embed-text` or similar | Semantic search over Obsidian vault — find notes by meaning, not just keyword |
| **Tools** | Any `tools/` script calling Ollama | Every tool that generates a response works instead of silently failing |
| **Dashboard** | Live status bar | Know Ollama is up without touching the terminal |
| **Future: n8n** | Add n8n container, point at same `host.docker.internal:11434` | Visual automation (Discord, email, webhooks) powered by local LLM — zero API cost |

| **Future: Custom Modelfiles** | `ollama create reclaw-agent -f Modelfile` | Bake specialized agents (grant-hunter brain, scraper brain) directly into Ollama |

## Why Local LLM Matters for This Project

Every call to OpenAI or Anthropic costs money and sends your data offsite. With Ollama on your own Hetzner box:

- **$0 per token** — run 10,000 agent calls or 1, same cost
- **Private** — Obsidian vault contents, grant research, business data never leaves the server
- **No rate limits** — agents can run loops, retries, and batch jobs without hitting API caps
- **Always available** — no outages, no deprecations, no model changes you didn't choose

The entire passive income stack — grant scanning, GBP monitoring, content generation, job aggregation — runs on hardware you control, at a fixed monthly Hetzner cost.
